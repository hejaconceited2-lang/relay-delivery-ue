# 接力送 · 分包分润方案

> 起草日期：2026-06-14
> 方案核心：平台方出资，运营方出力，利润五五分

---

## 一、模式定位

接力送业务从「直营」转为「平台分包」模式：

- **平台方（公司）**：出资、建点、对接美团，不介入日常运营
- **运营方（分包商）**：出人、管理、跑配送，不承担启动资金
- **分润逻辑**：每月核算净利润，双方各取 50%

本质是**资本合伙**——公司出钱，分包商出管理，风险共担、收益共享。

---

## 二、权责划分

### 平台方（公司）

| 阶段 | 责任 |
|------|------|
| **开设期** | 点位踩点、物业洽谈、合同签署 |
| | 物料采购与布置（货架、标识、扫码设备） |
| | 人员招聘支持、岗前培训、SOP 制定 |
| | 美团运力对接、系统账号开通 |
| **运营期** | 美团结算对账、收入归集 |
| | 系统维护、骑手关系管理 |
| | **按月垫付人力工资**（从收入池中优先扣除） |
| | 月度利润核算与分润打款 |

### 运营方（分包商）

| 阶段 | 责任 |
|------|------|
| **开设期** | 配合踩点，提供本地人力渠道 |
| | 按 SOP 完成岗前培训 |
| **运营期** | 人员招聘、排班、考勤管理 |
| | 每日配送执行（大堂接单→上楼送达） |
| | 服务质量保障（时效 ≤15min、完好率 ≥99%） |
| | 现场异常处理（骑手纠纷、物业沟通、丢件赔付） |
| | 按约定人效基准控制人力成本 |

## 三、财务模型

### 3.1 月度收入池（统一进入公司账户）

| 收入项 | 计算方式 | Phase 1（引流期） | Phase 2（长期补贴） | Phase 3（归零） |
|--------|---------|-------------------|--------------------|-----------------|
| 美团结算 | 月单量 × 结算价 | × ¥2.50 | × ¥2.50 | × ¥2.00 |
| 骑手费 | 月单量 × 费率 | × ¥1.00 | × ¥2.00 | × ¥2.00 |
| 人头补贴 | (T−1) × 80 × 30天 | 达标即享 | 达标即享 | 取消 |

> **补贴条件**：人均 ≥20 单/天 且 至少 1 人满 3 小时

### 3.2 月度成本（从收入池中扣除）

| 成本项 | 计算方式 | 月均金额 |
|--------|---------|---------|
| 人力成本 | 排班总工时 × ¥30/h × 30天 | 浮动 |
| 物料摊销 | ¥100/点位/月 | ¥100 |
| 丢件/赔付 | 实报实销（上限 ¥200/月） | ≤¥200 |

### 3.3 利润 & 分成

```
月度净利 = 美团结算 + 骑手费 + 人头补贴 − 人力成本 − 物料摊销 − 赔付

平台分成 = 月度净利 × 50%
运营方分成 = 月度净利 × 50%
```

- 月度净利 ≤0 时，运营方当月无分成，平台方承担全部亏损
- 连续 2 个月亏损，双方协商是否关停该点位

## 五、关键约束

### 5.1 人效基准（防磨洋工）

运营方控制排班，可能倾向多报人头——人头多了补贴涨，但人力成本也涨。需设定硬性标准：

| 指标 | 基准值 | 红线 |
|------|--------|------|
| 人均产能 | ≥12 单/h | <10 单/h 触发整改 |
| 人均单量 | ≥20 单/天 | <20 单/天 补贴归零 |

**实际人效低于 10 单/h 时，超出基准的人力成本从运营方分成中扣除。**

> 依据：6 个点位 571 单实测，成熟点位中位 12.9 单/h。

### 5.2 服务质量

| 指标 | 达标线 | 扣罚 |
|------|--------|------|
| 配送时效 | 15min 内完成率 ≥95% | 每低 1pp 扣月分成 ¥200 |
| 完好率 | ≥99% | 丢件/破损按外卖原价赔付，¥50 封顶/单 |
| 骑手投诉 | ≤2 次/月 | 超标的每次扣 ¥100 |

### 5.3 退出机制

- 连续 2 个月净利为负 → 双方协商关停，物料归平台方
- 运营方主动退出 → 提前 30 天通知，当月按实际运营天数结算
- 平台方主动关停 → 提前 15 天通知，当月全额结算

## 六、与直营模式的对比

| 维度 | 直营（当前） | 分包（本方案） |
|------|------------|--------------|
| 人力管理 | 平台直接招聘排班 | 运营方负责 |
| 成本结构 | 全部自担 | 平台垫付，从收入池扣回 |
| 规模化难度 | 高（每点需管理） | 低（借助本地人力渠道） |
| 单点利润 | 100% 归平台 | 50% 归运营方 |
| Phase 3 风险 | 全由平台承担 | 全由平台承担（运营方无分成） |
| 服务质量控制 | 直接管控 | 靠合同约束 |

**直营比分包多赚 50% 的净利润——前提是平台有能力直接管理所有点位。** 分包的核心价值不是省钱，是**用利润换规模**：借助当地人力渠道快速铺点，降低管理复杂度。

## 七、待决策问题

1. **人力成本垫付**：方案中公司垫付工资、从收入池扣除。是否改为运营方自行发薪？（给运营方更大的成本控制压力，但也提高了其资金门槛）

2. **补贴归属**：人头补贴本质是对冲人力成本的。是否将补贴全额归属运营方（作为对其人力投入的直接补偿），再从剩余利润中五五分？当前方案是统一入池。

3. **Phase 1 保护期**：引流期利润薄（40 单/天时双方仅各得 ¥400/月），是否设最低保底分成（如 ¥2,000/月/点）以吸引运营方入驻？

4. **多点打包**：一个运营方承包多个点位时，盈亏是否可以跨点抵消后再分润？还是逐点独立核算？
